introduction

Slim Starterkit is a plug-and-play, in-browser prototyping toolkit, created to jump-start high-fidelity prototyping. Slim Starterkit is NOT a framework, nor was it made for use in production. It's merely an attempt to bundle some helpful tools that I normally use in my workflow, as well as some top-notch default settings to ease the setup process.

rapid prototyping syntax

The syntax has been formulated for optimal typing speed. A class system based on common patterns for layout, spacing, positioning, typography and even common design patterns, like cards. Classnames are aptly named using an easy-to-memorize syntax, and most class functions are atomic in structure.

All positioning and spacing classes use a variable called $space which represents 1rem. The font-size is set to 100% on the body tag, meaning $space = 1rem = 16px.

Example
.mar-v_lg { margin-top:$space * 3;margin-bottom:$space * 3; } or .mar-v_lg { margin-top:48px;margin-bottom:48px; }

toolkit

This grunt task-set has everything you need to get started (static-file server, sass-compiler) set up based around the existing folder structure, so all you have to do is type in a few grunt commands, and you're ready to roll.

Features
  • Static file server - grunt-web-server
  • Sass Compiler - grunt-sass
  • Autoprefixer - Post-CSS
  • Browser refreshing / Style-injection - Browsersync
  • HTML templating system - grunt-bake
  • File Watch - grunt-contrib-watch

Directions
  1. Open Terminal
  2. Navigate to your project folder root
  3. Type `grunt server`. This will start your static file server
  4. Open another terminal window in the folder root and type `grunt`
  5. Start coding

With these two processes running, you can now code without worrying about refreshing the page, compiling your Sass, or prefixing your code. Follow the same process every time to spin up a server and start coding in no time.

Bonus

In addition to the grunt tasks, there is a small selection of even more useful tidbits baked right in:

  • Modernizr
  • jQuery
  • FontAwesome (via CDN)
  • Bootstrap (via CDN)
  • Angular (via CDN)

typography

Headings are scaled using a technique provided by Typecast. The default font-stack utilizes the reader's operating system's default font family.

The Stack
$default-font-stack:'San Francisco', 'Oxygen', 'Ubuntu', 'Roboto', 'Segoe UI', 'Helvetica Neue', 'Lucida Grande', sans-serif;
